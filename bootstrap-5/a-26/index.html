Barra de Navegação

  • O Bootstrap contém diversas classes para criar barras de navegação, melhorando seu visual em relação ao HTML padrão.

navbar:

  • Para a criação de uma Barra de Navegação utilizamos a TAG semântica do HTML 5 <nav>com a classe navbar.
  • Por padrão, a navbar assim como seu conteúdo são fluídos e para uma melhor responsividade, é indicado a colocação de um elemento container em seu interior para agrupar seu conteúdo.

<nav>

(navbar)

navbar-brand:

  • Para acrescentar o seu brand (marca) na barra de navegação, utilizamos um elemento de linha (<a>, <img>, <span>, etc) com a classe navbar-brand.
  • Para alinhamento do conteúdo da navbar-brand, transformamos o elemento de linha em elemento flex-box com a classe d-flex e aplicamos o alinhamento com as classes align-items-center (vertical) e/ou justify-content-center (horizontal).

<a> + icon-bootstrap

(navbar-brand)

navbar-text:

navbar-expand & navbar-collapse:

  • Para acrescentar um conjunto de elementos (links, botões, etc) para navegação do site, utilizamos um elemento de bloco (<div>) com as classes collapse + navbar-collapse e em seu interior outro elemento de bloco (<div>) com a classe navbar-nav que terá como conteúdo os links de navegação <a> com a classe nav-link.
  • Para que o conjunto de elementos (links) fiquem visíveis na barra de navegação é necessário incluírmos no elemento com a TAG <nav> e a classe navbar, a classe navbar-expand com um sufixo de breakpoint (sm, md, xl, xxl), fazendo com que na resolução escolhida seja mostrado.

<nav>

(navbar-expand-md & navbar-collapse)

navbar-toggle:

  • Podemos colocar um elemento <button> com type="button" e a classe navbar-toggler, logo acima do conjunto de elementos (links) para que quando a resolução esconda esses elementos, possa ser clicado para que os mesmos sejam revelados.
  • O <button> além da classe navbar-toggler deve conter os atributos data-bs-toggle com o valor collapse (data-bs-toggle="collapse") e data-bs-target com um valor, iniciado por #, que será o id do elemento (div) da classe collapse navbar-collapse e podendo inserir um ícone que represente o menu acrecentando um elemento de linha (span) com a classe navbar-togller-icon.

<nav>

(navbar-toggler & navbar-toggler-icon)

dropdown:

  • Podemos transformar cada elemento (link) da classe nav-link em um sub-menu suspenso envolvendo-o com um elemento de bloco (<div>) com a classe dropdown e acrescentando à classe nav-link a classe dropdown-toggle e o atributo data-bs-toggle com o valor dropdown (data-bs-toggle="dropdown") e logo abaixo do link acrescentar um elemento <ul> da classe dropdown-menu com elementos <li> com os links <a> com a classe dropdown-item.

<div>

posição do menu de navegação:

  • Para colocarmos o menu de navegação à direita da viewport, incluímos um margin automático à esquerda (ms-auto) no elemento de bloco (div) com a classe navbar-nav. Caso o menu suspenso do elemento de navegação mais à direita ultrapasse o limite da viewport, acrescenta-se a classe dropdown-menu-end à TAG <ul> com a classe dropdown-menu.

<menu de navegação>

stick-top:

  • Para fixarmos a navbar, assim como qualquer elemento de bloco no topo da página utilizamos a classe stick-top e para fixarmos na base da viewpot utilizamos a classe stick-bottom.
  • Caso o conteúdo da página não complete toda a altura da viewpot, o elemento fixado na base com a classe stick-bottom ficará no fim do conteúdo, não fixando no fim da viewpot. Para resolver este problema acrescentamos outro elemento de bloco do mesmo tamanho com visibility=none e a classe fixed-botton

<navbar>

(stick-top & stick-bottom)
